# 项目规划与落地清单（多协作）

目标：在多人协作下可持续推进上线、稳定性与扩展。

## 1) 里程碑

### M1 上线准备（P0）
- 生产环境变量齐全（VAPID/Kook/Sentry/Stripe/Sui/Redis/Cron）
- 生产数据库迁移完成
- Vercel 构建验证通过
- 合约主网部署并更新 `deployment.ts/metadata.json`

### M2 稳定性联调（P1）
- Web Push 联调
- Kook Bot 联调
- E2E 流程在预发布跑通
- 支付对账 cron + 告警链路验证

### M3 性能与容量（P2）
- 压测基线与报告
- 核心 API P95/P99 指标上线
- 告警阈值与告警通道确认

### M4 迭代扩展（P3）
- 搜索优化
- 实时聊天
- 多游戏扩展

## 2) 任务拆分（建议按模块分配）

- 前端：页面迭代、组件复用、体验优化
- 后端：API/服务层、权限与风控
- 链上：合约更新、同步与代付
- 小程序：Taro 端功能对齐
- 运维：环境变量、部署、监控与告警

## 3) 交付节奏（建议）

- 每周：固定一次版本窗口 + 回归
- 每日：任务同步与风险提示
- 每次发布：按 `docs/RELEASE_RUNBOOK.md` 执行

## 4) 风险点（持续跟踪）

- 订单/支付/链上为核心路径，改动需双人评审
- 统计与对账系统依赖数据准确性
- 环境变量缺失易导致运行时错误
