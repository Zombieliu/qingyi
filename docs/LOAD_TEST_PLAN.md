# 压测与容量基线计划（P2）

目标：建立系统吞吐与延迟基线，形成可重复的容量评估流程。

## 1) 范围与场景

优先覆盖高频与核心路径：
- 登录与会话：`/api/auth/session`
- 订单查询与创建：`/api/orders`
- 订单详情：`/api/orders/:id`
- 链上同步：`/api/orders/:id/chain-sync`（可选）
- 充值：`/api/pay`（可选）

## 2) 环境

- 目标环境：预发布（与生产配置接近）
- 数据规模：模拟真实数据量（订单/用户/支付记录）
- 缓存：Redis/Upstash 按生产配置

## 3) 指标

- P50/P95/P99 延迟
- 错误率（4xx/5xx）
- QPS 与并发
- CPU/内存/DB 连接/Redis 命中率

## 4) 基线定义

- 单接口 95% 请求 <= 500ms（可根据业务调整）
- 错误率 < 1%
- DB 连接池无持续饱和

## 5) 执行流程（建议）

1. 预热 3-5 分钟
2. 阶梯并发：10 → 50 → 100 → 200
3. 记录每个档位的 P95/P99 与错误率
4. 产出报告：瓶颈、优化项、可承载并发结论

## 6) 交付物

- 压测配置（脚本或工具配置文件）
- 报告（基线数据 + 优化建议）
