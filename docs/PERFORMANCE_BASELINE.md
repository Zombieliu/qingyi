# 性能指标与告警基线（P2）

目标：建立统一可观测口径，明确指标阈值与告警动作。

## 1) 核心指标

- API 延迟：P50/P95/P99
- 错误率：4xx/5xx 比例
- 支付成功率与对账差异
- SSE/Web Push 投递成功率
- DB 与缓存命中率

## 2) 建议阈值（可调）

- P95 延迟 > 1s：告警
- 5xx 错误率 > 1%：告警
- 支付对账缺失 > 0：告警
- Web Push 失败率 > 5%：告警

## 3) 数据来源

- Sentry（API 与前端）
- 业务埋点（GrowthEvent）
- 支付对账 cron 输出
- Redis/Upstash 监控

## 4) 告警通道

- 企业微信 / Kook
- Sentry Alert Rule

## 5) 落地步骤

1. 统计类 API 暴露 P95/P99（已有 Web Vitals 可复用）
2. 对账 cron 设置阈值并启用告警
3. 配置告警接收人/频道
