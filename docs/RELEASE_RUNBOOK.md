# 发布流程与回滚预案（P2）

目标：发布可控、可回滚，避免线上不可恢复问题。

## 1) 发布前检查

- `pnpm run lint`
- `pnpm run build`
- 关键路径自测（登录/下单/订单/充值）
- 如涉及 DB：确认迁移文件已提交

## 2) 发布步骤（Vercel）

1. 合并到 `main`
2. Vercel 触发构建
3. 观察构建日志与部署状态
4. 发布后冒烟测试

## 3) 数据库迁移

- 生产执行：`pnpm run db:deploy`
- 如果迁移有风险：在预发布先演练

## 4) 回滚方式

- 前端回滚：Vercel 回滚到上一个部署
- 后端逻辑回滚：回滚 commit + 重新部署
- DB 回滚：优先写向前迁移（避免强制回滚）

## 5) 发布后检查

- 监控告警是否触发
- 关键 API 是否正常
- 订单/支付/链上流程是否受影响
